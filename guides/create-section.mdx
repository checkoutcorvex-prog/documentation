---
title: "Create a Custom Corvex Section Step by Step"
description: "Step-by-step tutorial for building a reusable Corvex Store section with a JSON schema, Liquid template, scoped CSS, and optional JavaScript interactivity."
---

A section in Corvex Store is a self-contained, independently configurable building block that merchants can add, remove, reorder, and configure through the visual editor. Every section consists of at minimum two files: a `schema.json` that defines what a merchant can configure, and a `template.liquid` that produces the rendered HTML. This guide walks you through building a complete **Testimonials Grid** section — including blocks, scoped styles, and optional JavaScript interactivity.

## What you will build

A testimonials section where merchants add individual testimonials as blocks. Each block has a customer photo, name, role, quote, and a star rating. The section-level settings control the layout columns, padding, and colors.

```
sections/testimonials/
├── schema.json
└── template.liquid
```

<Steps>

<Step title="Define the schema">

The schema tells the editor what settings and blocks are available. Create `sections/testimonials/schema.json`:

```json
{
  "name": "Testimonials",
  "category": "content",
  "settings": [
    {
      "type": "header",
      "content": "Section heading"
    },
    {
      "type": "text",
      "id": "title",
      "label": "Title",
      "default": "What our customers say"
    },
    {
      "type": "select",
      "id": "title_alignment",
      "label": "Title alignment",
      "options": [
        { "value": "left",   "label": "Left"   },
        { "value": "center", "label": "Center" },
        { "value": "right",  "label": "Right"  }
      ],
      "default": "center"
    },
    {
      "type": "header",
      "content": "Appearance"
    },
    {
      "type": "color",
      "id": "background_color",
      "label": "Background color",
      "default": "#f9f9f9"
    },
    {
      "type": "color",
      "id": "card_background",
      "label": "Card color",
      "default": "#ffffff"
    },
    {
      "type": "select",
      "id": "columns",
      "label": "Columns",
      "options": [
        { "value": "2", "label": "2 columns" },
        { "value": "3", "label": "3 columns" }
      ],
      "default": "3"
    },
    {
      "type": "range",
      "id": "padding_vertical",
      "label": "Vertical padding (px)",
      "min": 20,
      "max": 120,
      "step": 10,
      "default": 60,
      "unit": "px"
    }
  ],
  "blocks": [
    {
      "type": "testimonial",
      "name": "Testimonial",
      "settings": [
        {
          "type": "image_picker",
          "id": "avatar",
          "label": "Customer photo"
        },
        {
          "type": "text",
          "id": "name",
          "label": "Customer name",
          "default": "Jane Smith"
        },
        {
          "type": "text",
          "id": "role",
          "label": "Role or context",
          "default": "Customer since 2023"
        },
        {
          "type": "textarea",
          "id": "text",
          "label": "Testimonial",
          "default": "Outstanding product — arrived quickly and exceeded expectations!"
        },
        {
          "type": "range",
          "id": "rating",
          "label": "Star rating",
          "min": 1,
          "max": 5,
          "step": 1,
          "default": 5
        }
      ]
    }
  ],
  "max_blocks": 12,
  "presets": [
    {
      "name": "3 testimonials",
      "blocks": [
        { "type": "testimonial", "settings": { "name": "Jane Smith",   "rating": 5 } },
        { "type": "testimonial", "settings": { "name": "John Adams",   "rating": 5 } },
        { "type": "testimonial", "settings": { "name": "Maria Garcia", "rating": 4 } }
      ]
    }
  ]
}
```

<AccordionGroup>
  <Accordion title="Always provide defaults for every setting">
    Every setting should have a `"default"` value. This ensures the section renders correctly before a merchant has configured anything, and prevents `null` values appearing in the HTML.
  </Accordion>
  <Accordion title="Use presets to give merchants a good starting point">
    Presets populate the section with sample blocks when a merchant first adds it from the editor panel. Without a preset, the section starts empty and may look broken.
  </Accordion>
  <Accordion title="max_blocks limits how many blocks a merchant can add">
    Set a `max_blocks` that reflects your layout. A 3-column grid with 12 testimonials fills 4 rows — a reasonable limit that keeps performance in check.
  </Accordion>
</AccordionGroup>

</Step>

<Step title="Create the Liquid template">

Create `sections/testimonials/template.liquid`. Scope all CSS rules to `section.id` so that two instances of this section on the same page never conflict.

<Warning>
The root `<section>` element **must** have `data-section-type="{{ section.type }}"` and `data-section-id="{{ section.id }}"`. Without these attributes the editor cannot select the section, and the JavaScript registry cannot initialize lifecycle hooks. See [Data Attributes](/sections/data-attributes).
</Warning>

```liquid
{% style %}
  .testimonials-{{ section.id }} {
    background-color: {{ section.settings.background_color }};
    padding: {{ section.settings.padding_vertical }}px 0;
  }

  .testimonials-{{ section.id }} .testimonials__title {
    font-family: var(--font-heading);
    font-size: clamp(1.5rem, 3vw, 2.5rem);
    margin: 0 0 48px;
    color: var(--color-text);
  }

  .testimonials-{{ section.id }} .testimonials__grid {
    display: grid;
    gap: 24px;
  }

  .testimonials-{{ section.id }} .testimonials__grid--2col {
    grid-template-columns: repeat(2, 1fr);
  }

  .testimonials-{{ section.id }} .testimonials__grid--3col {
    grid-template-columns: repeat(3, 1fr);
  }

  @media (max-width: 768px) {
    .testimonials-{{ section.id }} .testimonials__grid--2col,
    .testimonials-{{ section.id }} .testimonials__grid--3col {
      grid-template-columns: 1fr;
    }
  }

  .testimonials-{{ section.id }} .card {
    padding: 28px;
    border-radius: var(--border-radius);
    box-shadow: 0 2px 12px rgba(0,0,0,0.06);
  }

  .testimonials-{{ section.id }} .card__stars {
    margin-bottom: 16px;
    font-size: 1.2rem;
  }

  .testimonials-{{ section.id }} .star--active { color: #f5a623; }

  .testimonials-{{ section.id }} .card__quote {
    font-style: italic;
    margin: 0 0 20px;
    line-height: 1.7;
    color: #555;
  }

  .testimonials-{{ section.id }} .card__author {
    display: flex;
    align-items: center;
    gap: 12px;
  }

  .testimonials-{{ section.id }} .card__avatar {
    width: 48px;
    height: 48px;
    border-radius: 50%;
    object-fit: cover;
  }

  .testimonials-{{ section.id }} .card__info {
    display: flex;
    flex-direction: column;
    gap: 2px;
    font-size: 0.875rem;
  }
{% endstyle %}

<section class="testimonials testimonials-{{ section.id }}"
         data-section-type="{{ section.type }}"
         data-section-id="{{ section.id }}">
  <div class="container">

    {% if section.settings.title != blank %}
      <h2 class="testimonials__title"
          style="text-align: {{ section.settings.title_alignment }}">
        {{ section.settings.title }}
      </h2>
    {% endif %}

    {% if section.blocks.size > 0 %}
      <div class="testimonials__grid testimonials__grid--{{ section.settings.columns }}col">

        {% for block in section.blocks %}
          <div class="card"
               style="background-color: {{ section.settings.card_background }}"
               data-block="{{ block.type }}"
               data-block-id="{{ block.id }}">

            <div class="card__stars">
              {% for i in (1..block.settings.rating) %}
                <span class="star star--active">★</span>
              {% endfor %}
              {% assign remaining = 5 | minus: block.settings.rating %}
              {% for i in (1..remaining) %}
                <span class="star">☆</span>
              {% endfor %}
            </div>

            {% if block.settings.text != blank %}
              <blockquote class="card__quote">
                "{{ block.settings.text }}"
              </blockquote>
            {% endif %}

            <div class="card__author">
              {% if block.settings.avatar != blank %}
                <img src="{{ block.settings.avatar | img_url: '60x60' }}"
                     alt="{{ block.settings.name }}"
                     class="card__avatar"
                     width="48"
                     height="48"
                     loading="lazy">
              {% endif %}
              <div class="card__info">
                {% if block.settings.name != blank %}
                  <strong>{{ block.settings.name }}</strong>
                {% endif %}
                {% if block.settings.role != blank %}
                  <span>{{ block.settings.role }}</span>
                {% endif %}
              </div>
            </div>

          </div>
        {% endfor %}

      </div>
    {% else %}
      <p style="text-align: center; opacity: 0.5;">
        Add testimonials by clicking "Add block" in the sidebar.
      </p>
    {% endif %}

  </div>
</section>
```

</Step>

<Step title="Add the section to a page">

After creating the files, the `testimonials` section appears automatically in the editor's section panel. To add it programmatically:

```bash
curl -X POST https://api.corvex.io/api/stores/{storeId}/pages/{pageId}/sections \
  -H "Content-Type: application/json" \
  -d '{
    "type": "testimonials",
    "settings": {
      "title": "What our customers say",
      "columns": "3"
    },
    "order": 3
  }'
```

</Step>

<Step title="Add JavaScript interactivity (optional)">

The section above is 100% server-rendered. If you need interactive behavior — tabs, carousels, animated counters — extend `BaseSection` in TypeScript.

<Tabs>
  <Tab title="TypeScript class">
```typescript
// src/lib/sections/testimonials.ts
import { BaseSection, defineSection } from './base-section';

interface TestimonialsOptions {
  autoplay: boolean;
}

class TestimonialsSection extends BaseSection<TestimonialsOptions> {
  static readonly type = 'testimonials';

  private currentIndex = 0;

  mount() {
    this.delegate('click', {
      'prev': () => this.prev(),
      'next': () => this.next(),
    });

    if (this.options.autoplay) {
      const id = setInterval(() => this.next(), 4000);
      this.addCleanup(() => clearInterval(id));
    }
  }

  private prev() {
    this.currentIndex--;
    this.render();
  }

  private next() {
    this.currentIndex++;
    this.render();
  }

  private render() {
    const slides = this.refs<HTMLElement>('slide');
    const active = ((this.currentIndex % slides.length) + slides.length) % slides.length;
    slides.forEach((s, i) => s.hidden = i !== active);
  }
}

defineSection(TestimonialsSection);
```
  </Tab>
  <Tab title="Updated template">
```liquid
<section class="testimonials testimonials-{{ section.id }}"
         data-section-type="{{ section.type }}"
         data-section-id="{{ section.id }}"
         data-autoplay="{{ section.settings.autoplay }}">

  {% for block in section.blocks %}
    <div class="card"
         data-ref="slide"
         data-block="{{ block.type }}"
         data-block-id="{{ block.id }}">
      ...card content...
    </div>
  {% endfor %}

  <button data-action="prev">← Previous</button>
  <button data-action="next">Next →</button>

</section>
```
  </Tab>

<Warning>
The `type` value must match exactly in four places: the folder name, `data-section-type` in the template, `static readonly type` in the TypeScript class, and the key in `SECTION_SCRIPT_MAP`. A mismatch causes the JavaScript bundle to never load — the section renders but clicks do nothing.
</Warning>

<Tip>
If clicks are not responding, open the preview iframe console and run `window.CorvexSections.getDebugInfo()`. If `testimonials` does not appear in `registeredTypes`, the bundle did not load — check the bundle registration steps above.
</Tip>

## Best practices

<AccordionGroup>
  <Accordion title="Always scope CSS with section.id">
    Without the `section.id` prefix, two instances of the same section on the same page produce CSS conflicts. Use `.my-section-{{ section.id }} .element { ... }` for every rule in `{% style %}`.
  </Accordion>
  <Accordion title="Handle empty states explicitly">
    Always check `section.blocks.size > 0` before iterating blocks, and `!= blank` before rendering text fields. An unconfigured section should display a clear placeholder, not a broken layout.
  </Accordion>
  <Accordion title="Use this.addCleanup() for timers and global listeners">
    In the editor, sections remount on every setting change. Without cleanup, each update creates an additional timer or event listener. `this.addCleanup()` and `this.on()` handle this automatically.
  </Accordion>
  <Accordion title="Test with two instances side by side">
    The most common isolation bugs only appear with two instances of the same section on one page. Before shipping, add the section twice with different settings and confirm they behave independently.
  </Accordion>
  <Accordion title="Write mobile styles first">
    Write base styles for small screens and use `@media (min-width: ...)` to enhance for larger ones. Merchants can place sections on any page at any breakpoint.
  </Accordion>
</AccordionGroup>
